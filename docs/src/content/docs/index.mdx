## Why Runjucks?

<CardGrid stagger>
	<Card title="Nunjucks-shaped API" icon="star">
		`renderString`, `Environment`, `configure`, `compile`, and `Template` follow the same ideas as [Nunjucks](https://mozilla.github.io/nunjucks/). See [JavaScript API](guides/javascript-api/).
	</Card>
	<Card title="Rust-powered engine" icon="rocket">
		Templates are lexed, parsed, and rendered in Rust — no compile-to-JavaScript step or `eval` on the hot path. Custom delimiters, trim markers, and whitespace options match Nunjucks-style behavior.
	</Card>
	<Card title="TypeScript" icon="document">
		Types ship with the package (`index.d.ts`). The full Node API reference is [generated on build](api/).
	</Card>
	<Card title="Tested against Nunjucks" icon="approve-check">
		Golden fixtures and parity checks help keep behavior aligned with the `nunjucks` npm package. Contributors: see [Development](guides/development/) and the repo’s parity notes.
	</Card>
	<Card title="Performance" icon="rocket">
		Reuse environments and `Template` instances, ship release native builds, and understand built-in parse caching — see [Performance](guides/performance/).
	</Card>
</CardGrid>

### What you get

**Template language:** control flow (`if`, `for`, `switch`), `set`, `include`, `extends`, `block`, `import` / `from`, `macro`, `call` / `caller`, `filter` blocks, `raw` / `verbatim`, and rich `{{ }}` expressions with filters and `is` tests — see [Template language](guides/syntax/).

**Node API:** environment options (autoescape, `throwOnUndefined`, `trimBlocks`, `lstripBlocks`, custom delimiters), in-memory template maps, custom filters, tests, tag extensions, and globals — see [JavaScript API](guides/javascript-api/) and [Limitations](guides/limitations/).
