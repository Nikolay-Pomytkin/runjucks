# Runjucks documentation site

Built with [Astro Starlight](https://starlight.astro.build/). The **Node.js API** pages are generated by [TypeDoc](https://typedoc.org/) from the parent [`index.d.ts`](../index.d.ts) into `public/api/`. **Rust API** HTML is generated with `cargo doc` on **`runjucks_core`** into `public/rustdoc/` (served under `/rustdoc/runjucks_core/`).

**What works today?** See the [Syntax and parity](src/content/docs/guides/syntax.md) guide (supported `{{ }}`, lexer tags, conformance skips). The site uses a small [custom CSS](src/styles/custom.css) layer and `public/favicon.svg`.

## Node version

This package follows **Astro’s** current requirement: **Node.js ≥ 22.12** (see [`package.json`](./package.json) `engines`). Use the same major in CI and locally.

## Commands

| Command | Description |
|---------|-------------|
| `npm run dev` | Local dev server |
| `npm run api` | Regenerate TypeDoc only |
| `npm run api:rust` | Regenerate rustdoc into `public/rustdoc/` (requires Rust toolchain) |
| `npm run build` | TypeDoc + rustdoc + static site → `dist/` |

### Optional Bun

From this directory you can run **`bun install`**, **`bun run dev`**, **`bun run build`**, **`bun run api`**, **`bun run api:rust`** instead of npm. CI still uses **Node + npm**; if anything fails under Bun, use Node.

## GitHub Pages

The workflow [`.github/workflows/docs.yml`](../.github/workflows/docs.yml) builds for the live URL:

**https://nikolay.pomytkin.com/runjucks/**

It sets `ASTRO_SITE=https://nikolay.pomytkin.com` and `ASTRO_BASE_PATH=/runjucks/` so Astro emits correct canonical URLs, sitemap, and asset paths. Configure your **GitHub Pages** custom domain and DNS to serve that host (or a reverse proxy) as usual.

## Local preview with a base path

Match production:

```bash
ASTRO_BASE_PATH=/runjucks/ ASTRO_SITE=https://nikolay.pomytkin.com npm run build
npm run preview
```

For a quick local build at the site root (no subpath), omit the env vars or use `ASTRO_BASE_PATH=/`.
